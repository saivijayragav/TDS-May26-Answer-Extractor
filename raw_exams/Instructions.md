# Reverse Engineering TDS Exams

This directory contains the raw `exam.mjs` files downloaded directly from the TDS course platform. When a new Graded Assignment (GA) is released, you can use the instructions below to reverse-engineer its logic and add it to the Answer Extractor web app.

## 1. Anatomy of an `exam.mjs` File

The `exam.mjs` file is an ES Module bundle (usually generated by a tool like esbuild or Webpack). 
At its core, it exports a single main function, usually called `questions`, which takes a `user` object containing an email address:

```javascript
export async function questions({ user: l, weight: u = 1 }) {
    // ...
    return {
        "q-some-id": await SomeQuestionLogic({ user: l, weight: u }),
        "q-another-id": await AnotherQuestionLogic({ user: l, weight: u }),
    };
}
```

Each question function returns an object structured roughly like this:
```javascript
{
    id: "q-unique-id",
    title: "Question Title",
    question: litHtmlTemplateString,
    answer: "string" | function
}
```

## 2. Categorizing Questions

To build an extractor, you must first locate the question functions in the file and categorize them into three types based on the `answer` property:

1. **Deterministic (`Auto`)**: The answer is a fixed primitive value (String, Number, Hash) that is generated entirely by pseudo-random logic seeded with the user's email.
2. **Semi-Deterministic (`Semi`)**: The script validates a specific action (like an HTTP POST or an SQL query). You cannot just extract a string; you must extract the *parameters* (like a `request_id`) or provide a valid *command* (like a `SELECT` statement).
3. **Manual**: The answer involves a regex validator for a CLI command, uploading an asciinema cast, submitting a Markdown document, or using LLM verification. For these, we provide manual step-by-step instructions.

## 3. Extracting the Logic (Deterministic Questions)

Almost all dynamic questions in the exam use a seeded pseudo-random number generator (PRNG) to ensure each student gets unique data. The PRNG is typically initialized with the student's email and the question ID:

```javascript
let n = (0, seedrandom_default)(`${user.email}#${question_id}`);
```

### Steps to Extract:
1. **Locate the seed:** Find the string used to initialize the PRNG (e.g., `${email}#q-multi-cursor-json`).
2. **Trace the PRNG calls:** Replicate the exact sequence of PRNG calls (`n()`). **The order matters immensely!** If the exam script evaluates random names, then random timestamps, then random settings, your extraction script must execute the PRNG in the exact same sequence.
3. **Replicate the logic:** Translate the bundled, minified Javascript back into readable code in `js/gaX.js`. Use the helpers provided in `utils.js` (like `pick`, `randInt`, `shuffle`, `blobHash`, `treeHash`).

**Important Note on Asynchronous Logic:**
In some complex questions (like Git Time Travel), the exam script might generate an array of objects in one loop, and then loop over them *again* later to assign more random properties (like environment variables). You must study the closures carefully to ensure your PRNG sequence matches perfectly.

## 4. Extracting the Logic (Semi-Deterministic Questions)

If the `answer` is a closure (a function), you need to look at what it validates.

**Example 1: HTTP Post**
If the validator expects a JSON string containing a specific `request_id`, find where the `request_id` is generated. Your extractor should generate the `request_id` and output a JSON string formatted exactly as the validator expects.

**Example 2: SQL Average**
If the validator runs `c.exec(answer)` against an in-memory SQLite database, the answer *must* be a valid SQL query. Your extractor should provide a correct SQL query that solves the problem.

## 5. Adding to the Web App

Once you have analyzed a new exam file (e.g., GA2):

1. Create a new file `js/ga2.js`.
2. Define `window.GA2_QUESTIONS = [...]` following the schema used in `ga1.js`:
   ```javascript
   window.GA2_QUESTIONS = [
     {
       number: 1,
       id: "q-id",
       title: "Question Title",
       type: "deterministic", // or "semi", or "manual"
       async extract(email) {
          const rng = window.makePRNG(`${email}#q-id`);
          // ... logic ...
          return { answer: "result", note: "Optional note" };
       }
     }
     // ...
   ];
   ```
3. Add a script tag in `index.html`:
   ```html
   <script src="js/ga2.js"></script>
   ```
4. Register it in `js/app.js`:
   ```javascript
   const GA_REGISTRY = {
     1: { label: "GA1 – Tools in Data Science", questions: window.GA1_QUESTIONS },
     2: { label: "GA2 – ...", questions: window.GA2_QUESTIONS },
   };
   ```

The UI will automatically update to include the new GA in the dropdown!
